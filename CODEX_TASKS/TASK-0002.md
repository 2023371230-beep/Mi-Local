# TASK-0002 — Agente PRO fase B: bucle iterativo

## Objetivo
El agente debe: (a) leer contenido de archivos clave al planear (no solo el arbol) y
(b) tras ejecutar un comando aprobado, alimentar stdout/stderr al LLM para proponer el
siguiente paso — manteniendo aprobacion humana en CADA paso.

## Archivos involucrados
- backend/app/application/agent/agent_service.py
- backend/app/domain/schemas.py (si hace falta AgentStep nuevo tipo "followup")
- backend/app/presentation/endpoints/agent.py (endpoint POST /agent/sessions/{id}/next)
- frontend/components/agent/* (boton "Proponer siguiente paso" tras output de comando)

## Comportamiento esperado
1. plan(goal): ademas del arbol, leer hasta 5 archivos relevantes (heuristica: mencionados
   en goal, o README/package.json/pyproject) con SafetyPolicy.validate_read; max 4k chars c/u.
2. next_step(session): toma el ultimo output de comando/test + plan + estado y pide al LLM
   UN paso siguiente (schema Pydantic con format=json_schema de Ollama). Nunca auto-ejecuta.
3. Cada paso propuesto entra al mismo flujo de aprobacion existente.
4. Registrar en actions.log y persistencia .ai-local ya existente.

## Restricciones
- NO full-auto. NO ampliar whitelist de comandos en esta tarea (eso es TASK fase C).
- Prompt injection: contenido de archivos va marcado como datos no confiables (ya hay
  patron en _EDIT_SYSTEM_PROMPT; replicarlo).

## Pruebas
- test_agent_next_step: con LLM fake, output de comando fallido genera paso de correccion;
  paso requiere aprobacion; sin aprobacion no se ejecuta.
- Todos los tests existentes del agente siguen verdes.

## Criterios de aceptacion
- Flujo comando→output→propuesta→aprobacion funciona end-to-end en /agent con backend vivo.
- lint+build frontend ok; pytest ok.

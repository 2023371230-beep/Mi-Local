# TASK-0003 — Router fino (F6) + RAG contextual en skills

## Objetivo
Reducir falsos positivos del router regex y hacer que las skills tematicas citen documentos
del RAG cuando haya material relevante.

## Archivos involucrados
- backend/app/application/router/routing_rules.py
- backend/app/application/router/query_router.py
- backend/app/application/skills/base_skill.py (+ skills tematicas)
- backend/tests/test_router.py, test_skills.py

## Comportamiento esperado
1. Podar patrones sobre-anchos: quitar \berror\b, \bapi\b, \blogs?\b de las listas
   genericas; exigir 2+ matches o patrones compuestos para clasificar.
2. Segunda pasada LLM: si NINGUNA regex matchea y mode=auto, clasificar con llama3.2
   (prompt corto, respuesta de una palabra, timeout 5s, fallback a general).
3. skill_ui_ux debe usar ollama_general_model para prosa (coder solo si el mensaje pide
   codigo — heuristica: contiene "codigo|componente|tsx|css").
4. RAG contextual: en skills tematicas, mini-query a su coleccion (top_k=3); si distancia
   minima <= rag_max_distance, anexar chunks al prompt y devolver sources con rag_used=true.
   Si no, comportamiento actual sin RAG.

## Restricciones
- No cambiar contratos de ChatRequest/ChatResponse (rag_used/sources ya existen).
- Latencia extra maxima aceptable: +1.5s p50 (medir con logs).

## Pruebas
- Casos: "logs de mi app Next" NO va a ciberseguridad; "optimiza mi query de Chroma" NO va
  a bases_datos por la palabra index; "que es OWASP" SI va a ciberseguridad.
- RAG contextual: con vector client fake, distancia buena => sources presentes;
  distancia mala => sin sources.

## Criterios de aceptacion
- test_router ampliado verde; eval_rag 20/20; sin regresion en test_skills.

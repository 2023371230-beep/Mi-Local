# TASK-0004 — URGENTE: convertir frontend de gitlink a carpeta del monorepo

## Problema
frontend/ es un repositorio git anidado. El repo raiz lo registra como gitlink (modo 160000)
sin .gitmodules => en GitHub main NO existe el codigo del frontend. El backend si esta respaldado.

## Verificacion previa (ejecutar y pegar salida)
cd "C:\Users\angel\Modelo local\modelo-ia-carrera"
git ls-files --stage frontend        # esperar 160000 commit <hash>
git -C frontend log --oneline -5     # anotar hashes: NO PERDERLOS

## Procedimiento (sin force push, sin borrar codigo)
1. Backup del historial anidado:
   git -C frontend bundle create ..\frontend-nested-history.bundle --all
   (el bundle queda FUERA del indice; NO commitearlo)
2. Quitar el gitlink del indice raiz:
   git rm --cached frontend
3. Retirar la metadata git interna (NO el codigo):
   Rename-Item frontend\.git frontend\.git-backup-nested
   (tras verificar el push, puede borrarse; mientras, el root .gitignore no lo cubre:
   agregar linea "frontend/.git-backup-nested/" a .gitignore)
4. Verificar ignores ANTES de add: git status debe NO listar node_modules, .next,
   .env.local, test-results (el .gitignore raiz ya fue parcheado con .env.*).
   Comprobar: git check-ignore frontend/.env.local frontend/node_modules -v
5. git add frontend .gitignore
6. Validar: cd frontend; npm run lint; npm run build; npm run test:ui (con dev server si aplica)
7. Commit atomico: "fix(repo): frontend como carpeta del monorepo (antes gitlink anidado)"
8. git push origin main
9. Verificar: git ls-files frontend | Measure-Object -Line  (esperar >150 archivos)
   y confirmar en GitHub que frontend/package.json existe en main.

## Criterios de aceptacion
- git ls-files --stage frontend ya no muestra 160000.
- GitHub main contiene frontend/ completo sin node_modules/.next/.env.local.
- lint+build verdes antes del commit.
- Bundle de respaldo del historial anidado existe.
